# Piuda 기능명세서

> 치매 환자를 돌보는 **보호자**와 **간병인**을 연결하는 케어 관리 서비스

---

## 1. 사용자 (User)

### 역할 구분
| 역할 | 설명 |
|------|------|
| 보호자 (PROTECTOR) | 치매 환자의 가족. 환자를 직접 등록하고 간병인을 초대함 |
| 간병인 (CAREGIVER) | 전문 간병인. 초대코드를 통해 환자 케어팀에 합류함 |

### 기능 목록
| 기능 | 설명 |
|------|------|
| 회원가입 | 이메일/비밀번호/이름/전화번호/역할 입력. 간병인은 경력 연수·소개 추가 입력 |
| 로그인 | 이메일+비밀번호 인증 → JWT 토큰 발급 (유효시간 30분) |
| 내 정보 조회 | 현재 로그인한 사용자 정보 반환 (이름, 이메일, 역할, 내공점수 포함) |
| 내 정보 수정 | 이름·전화번호·비밀번호 변경 가능 |
| 회원 탈퇴 | 계정 영구 삭제 |
| 내공점수 랭킹 | 점수 높은 순으로 상위 N명 목록 조회 |

### 내공점수 시스템
- 가입 시 기본 **100점** 지급
- 커뮤니티에서 작성한 댓글이 채택되면 **+10점**
- 채택이 취소되면 **-10점** (최소 0점)
- 점수 기반 랭킹 제공 (Naver 지식인 방식)

## 2. 환자 (Patient)

### 기능 목록
| 기능 | 접근 권한 | 설명 |
|------|-----------|------|
| 환자 등록 | 보호자 | 이름·생년월일·성별·치매 단계 입력. 등록 시 8자리 초대코드 자동 생성 |
| 초대코드로 합류 | 간병인·보호자 | 보호자가 공유한 초대코드 입력 → 해당 환자 케어팀 참여 |
| 내 환자 목록 | 로그인 사용자 | 본인이 관리 중인 환자 전체 목록 |
| 환자 상세 조회 | 케어팀 구성원 | 환자 기본 정보 + 초대코드 조회 |
| 환자 정보 수정 | 케어팀 구성원 | 이름·생년월일·성별·치매 단계 수정 |
| 환자 삭제 | 케어팀 구성원 | 환자 레코드 삭제 |
| IoT 디바이스 연동 | 케어팀 구성원 | 디바이스 시리얼 입력 → 환자에 연결 |
| IoT 디바이스 연동 해제 | 케어팀 구성원 | 환자-디바이스 연결 해제 |

> **초대 흐름**: 보호자가 환자 등록 → 앱에서 초대코드 확인 → 간병인에게 코드 공유 → 간병인이 코드 입력하여 합류

## 3. 환자 신상 정보 (Patient Memory)

환자 1인당 1개의 의료·개인 정보 레코드. 환자 등록 시 빈 레코드 자동 생성.

| 항목 | 예시 |
|------|------|
| 혈액형 | A형 |
| 장기요양 등급 | 2등급 |
| 치매 유형 | 알츠하이머 |
| 동반 질환 | 고혈압, 당뇨 |
| 복용 약물 | 혈압약 1일 1회 |
| 금기 사항 | 없음 |
| 좋아하는 것 | 트로트, 화투 |
| 싫어하는 것 | 큰 소리 |
| 진정 효과 있는 말 | 잘 하셨어요 |
| 역효과 나는 말 | 왜 또 그러세요 |
| 석양증후군 정보 | 저녁 6시 이후 불안 증가 |
| 반복 행동 | 집에 가겠다고 반복 |
| 배회 경로 | 현관 → 거실 반복 |
| 비상 연락처 | 아들 010-xxxx-xxxx |
| 특이사항 | 물 자주 드셔야 함 |

- 조회·수정만 가능 (삭제 없음, 환자 삭제 시 함께 삭제)

## 4. 하루 일지 (Daily Log)

간병인 또는 보호자가 하루 케어 내용을 기록하는 기능.

### 기록 항목
| 분류 | 항목 |
|------|------|
| 신체 지원 | 위생, 목욕, 식사 도움, 체위 변경, 이동 도움, 화장실 도움, 총 시간 |
| 인지 지원 | 인지 자극, 생활 함께하기, 행동 관리 시간 (분) |
| 정서 지원 | 정서 소통 시간 — **간병인만 기입 가능** |
| 가사 지원 | 식사 청소, 개인 도움, 총 시간 |
| 건강 추이 | 신체 기능 추이 (IMPROVING / STABLE / DECLINING), 식사 기능 추이 |
| 배변 기록 | 대변·소변 실금 횟수 |
| 특이사항 | 자유 텍스트 |
| 사진 | 선택적 이미지 첨부 |

### 연동 기능
- 일지 작성 시 **케어 캘린더에 자동으로 일지 도장** 생성 (DAILY_LOG 타입)
- 일지 삭제 시 연동된 캘린더 항목도 함께 삭제

## 5. 케어 캘린더 (Care Calendar)

케어팀이 공유하는 일정 관리 기능.

| 기능 | 설명 |
|------|------|
| 일정 등록 | 제목·내용·카테고리·시작/종료 시간·담당자 지정 |
| 일정 목록 조회 | 특정 환자의 전체 일정 (MANUAL + DAILY_LOG 타입 통합) |
| 일정 수정 | 작성자 본인만 수정 가능 |
| 일정 삭제 | 작성자 본인만 삭제 가능 |

### 캘린더 타입
| 타입 | 생성 방식 |
|------|-----------|
| SCHEDULE | 사용자가 직접 등록 |
| DAILY_LOG | 하루 일지 작성 시 자동 생성 |

### 일정 카테고리
`OUTING`(외출) / `VISIT`(병원 방문) / `SUPPLY`(용품 구매) / `EVENT`(이벤트) / `ETC`(기타)

---

## 6. 기억 갤러리 (Memory Gallery)

환자 관련 사진과 IoT 디바이스가 녹음한 음성을 모아보는 공간.

| 기능 | 설명 |
|------|------|
| 사진 업로드 | 케어팀 구성원이 사진 + 메모 등록 (최대 10MB, jpg/png/gif/webp) |
| 갤러리 조회 | 일지 첨부사진 + 직접 업로드 사진 + IoT 음성 통합, 최신순 정렬 |
| 사진 삭제 | 업로드한 항목 삭제 (케어팀 구성원만) |

---

## 7. 커뮤니티 (Community)

보호자와 간병인 모두가 정보를 공유하는 게시판.

### 게시글
| 기능 | 설명 |
|------|------|
| 게시글 작성 | 제목·내용·카테고리 + 선택적 이미지 첨부 |
| 게시글 목록 | 전체 또는 카테고리 필터 조회, 최신순 |
| 게시글 상세 | 상세 내용 + 내 좋아요 여부 |
| 게시글 수정 | 작성자 본인만 |
| 게시글 삭제 | 작성자 본인만 |
| 좋아요 | 토글 방식 (누르면 추가, 다시 누르면 취소) |

### 카테고리
`TIP`(케어 팁) / `QUESTION`(질문) / `SHARE`(정보 공유) / `ETC`(기타)

### 댓글 & 채택
| 기능 | 설명 |
|------|------|
| 댓글 작성 | 로그인한 사용자 누구나 |
| 댓글 수정 | 작성자 본인만 |
| 댓글 삭제 | 작성자 본인만 |
| **댓글 채택** | 게시글 작성자가 가장 도움된 댓글 1개 선택 → 댓글 작성자에게 내공점수 +10 |
| **채택 취소** | 게시글 작성자가 채택 취소 → 내공점수 -10 |

## 8. AI 케어 조언 (Care Advice)

환자 정보를 기반으로 GPT가 개인화된 케어 조언을 제공하는 채팅 기능.

| 기능 | 설명 |
|------|------|
| 세션 생성 | 특정 환자 기준으로 AI 대화 세션 시작 |
| 메시지 전송 | 질문 입력 → AI가 환자 정보 컨텍스트를 반영하여 답변 |
| 세션 목록 조회 | 본인이 생성한 세션 전체 |
| 메시지 목록 조회 | 특정 세션의 대화 전체 |
| 세션 삭제 | 본인 세션만 삭제 가능 |

> **AI 컨텍스트**: 환자 이름, 생년월일, 성별, 치매 단계, 치매 유형, 동반 질환, 복용 약물, 성격 특성 등 신상 정보를 자동으로 GPT에게 전달하여 개인화된 조언 생성

---

## 9. IoT 디바이스 (Device)

ESP32 기반 IoT 기기를 통한 음성 기록 기능.

| 기능 | 설명 |
|------|------|
| 디바이스 등록 | 시리얼 번호로 디바이스 등록 (인증 불필요) |
| 음성 기록 업로드 | 디바이스가 녹음된 음성 파일 URL을 서버에 전송 |
| 환자 연동 | 보호자/간병인이 디바이스를 특정 환자에 연결 |
| 연동 해제 | 환자-디바이스 연결 해제 |

## 기술 스택

| 영역 | 기술 |
|------|------|
| 언어 | Java 21 |
| 프레임워크 | Spring Boot 3.5.0 |
| 인증 | Spring Security + JWT (Stateless, 30분) |
| DB | MySQL (`dementia_project` 스키마) |
| ORM | Spring Data JPA + Hibernate |
| 파일 스토리지 | AWS S3 (이미지, 음성) |
| AI | OpenAI GPT API |
| 캐시 | Spring Cache (인메모리, 랭킹 최적화) |
