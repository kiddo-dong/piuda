# Piuda 사용자 시나리오

> **서비스 한 줄 설명**: 치매 환자를 돌보는 보호자와 간병인이 함께 케어 정보를 공유하고 AI 조언을 받는 서비스

---

## 등장인물

| 인물 | 역할 | 설명 |
|------|------|------|
| 홍보호자 | 보호자 (PROTECTOR) | 치매 어머니를 모시는 아들. 환자를 등록하고 케어팀을 관리 |
| 김간병인 | 간병인 (CAREGIVER) | 전문 간병인. 평일 낮 시간 어머니를 돌봄 |

---

## 시나리오 1. 서비스 시작 — 환자 등록 및 케어팀 구성

> **목표**: 홍보호자가 어머니를 등록하고, 간병인 김간병인을 초대

**1단계 — 보호자 회원가입**
- 홍보호자가 앱에서 회원가입
  - 이메일: `protector@test.com` / 역할: 보호자(PROTECTOR)

**2단계 — 환자 등록**
- 로그인 후 "환자 등록" 탭에서 어머니 정보 입력
  - 이름: 이순자 / 생년월일: 1945-03-10 / 성별: 여성 / 치매 단계: 경증
- 등록 완료 후 **초대코드 `A1B2C3D4`** 자동 생성됨
- 홍보호자가 초대코드를 카카오톡으로 김간병인에게 전달

**3단계 — 간병인 회원가입 및 합류**
- 김간병인이 앱에서 회원가입
  - 역할: 간병인(CAREGIVER) / 경력: 3년
- "초대코드 입력" 화면에서 `A1B2C3D4` 입력
- 이순자 어머니의 케어팀에 합류 완료

---

## 시나리오 2. 일상 케어 — 하루 일지 작성

> **목표**: 김간병인이 하루 케어 내용을 기록하고, 보호자가 확인

**오전 — 케어 시작**
- 김간병인이 출근 후 어머니 케어 시작 (09:00)

**오후 — 일지 작성**
- 김간병인이 앱에서 "하루 일지 작성"
  - 신체 지원: 위생 O, 식사 도움 O, 이동 도움 O (총 60분)
  - 인지 자극: 30분 진행
  - 정서 소통: 20분 (간병인만 기입)
  - 건강 추이: 신체 기능 안정 / 식사 기능 호전
  - 특이사항: "점심 식사를 평소보다 잘 드셨음. 오후에 산책 30분"
  - 사진: 점심 식사 사진 첨부

**자동 연동**
- 일지 저장과 동시에 **케어 캘린더에 "김간병인님의 하루 일지 작성 완료" 일정 자동 등록**

**저녁 — 보호자 확인**
- 홍보호자가 퇴근 후 앱에서 오늘 일지 확인
- 캘린더 화면에서 오늘 케어 완료 표시 확인

---

## 시나리오 3. 환자 정보 관리 — 신상 정보 등록

> **목표**: 케어팀이 어머니의 의료·성격 정보를 앱에 기록해 AI 조언 품질 향상

- 홍보호자가 "환자 신상 정보" 탭 접속
- 기본 의료 정보 입력
  - 혈액형: A형 / 장기요양 등급: 2등급
  - 치매 유형: 알츠하이머 / 동반 질환: 고혈압
  - 복용 약물: 혈압약 1일 1회
- 개인 성격 정보 입력
  - 좋아하는 것: 트로트, 화투
  - 싫어하는 것: 큰 소리
  - 진정 효과 있는 말: "잘 하셨어요, 대단해요"
  - 역효과 나는 말: "왜 또 그러세요"
  - 석양증후군: 저녁 6시 이후 불안 증가
  - 배회 경로: 현관 → 거실 반복

---

## 시나리오 4. AI 케어 조언 요청

> **목표**: 김간병인이 어머니의 식사 거부 상황에서 AI에게 맞춤 조언 요청

**상황**: 점심시간 어머니가 밥을 전혀 드시려 하지 않음

- 김간병인이 앱에서 "AI 케어 조언" 탭 접속
- 새 대화 세션 시작 (어머니 이순자 기준)
- 질문 입력: "치매 어르신이 갑자기 밥을 안 드시려 해요. 어떻게 해야 할까요?"

**AI 응답 예시** (어머니의 신상 정보 반영)
> "이순자 어르신은 트로트를 좋아하시니, 식사 중 좋아하는 음악을 틀어드리면 식사 분위기를 만드는 데 도움이 될 수 있습니다. 또한 식욕이 없을 때는 평소 좋아하시는 반찬을 소량 준비하거나..."

- 대화를 이어가며 추가 상황 질문 가능
- 이전 대화 히스토리 유지 (최근 10개 메시지 컨텍스트 반영)

---

## 시나리오 5. 케어 캘린더 — 병원 일정 등록

> **목표**: 홍보호자가 다음 주 병원 방문 일정을 등록하고 김간병인에게 담당 지정

- 홍보호자가 "캘린더" 탭에서 "일정 추가"
  - 제목: 서울대병원 정기 검진
  - 카테고리: 병원 방문 (VISIT)
  - 시작: 2026-06-05 오전 10:00
  - 종료: 2026-06-05 오후 12:00
  - 담당자: 김간병인 지정

- 김간병인이 캘린더 화면에서 본인이 담당인 일정 확인

---

## 시나리오 6. 기억 갤러리 — 추억 사진 보관

> **목표**: 케어 중 찍은 사진을 모아 나중에 회고

- 산책 중 김간병인이 어머니와 찍은 사진을 앱 업로드
  - "오늘 날씨가 좋아서 한강 공원 산책 30분" 메모 입력
- 홍보호자가 갤러리 탭에서 확인
  - 일지 첨부 사진 + 직접 업로드 사진 + IoT 음성 녹음 통합 목록 최신순 확인

---

## 시나리오 7. 커뮤니티 — 케어 팁 공유 및 내공점수 획득

> **목표**: 보호자가 질문을 올리고, 간병인이 답변하여 내공점수 획득

**질문 등록 — 홍보호자**
- "커뮤니티" 탭에서 글 작성
  - 카테고리: 질문(QUESTION)
  - 제목: "치매 어르신이 밤에 자꾸 일어나실 때 어떻게 하세요?"
  - 내용 입력 후 등록

**답변 작성 — 김간병인**
- 게시글 확인 후 댓글 작성
  - "저는 낮 시간 활동량을 늘리고 저녁에 따뜻한 우유를 드리는 것이 효과 있었습니다."

**채택 — 홍보호자**
- 댓글이 도움이 되었다고 판단 → "채택" 버튼 클릭
- 김간병인 내공점수: 100점 → **110점**

**랭킹 확인**
- 랭킹 탭에서 김간병인이 상위 랭크로 올라온 것 확인

---

## 시나리오 8. IoT 디바이스 연동 — 음성 기록 저장

> **목표**: ESP32 디바이스가 어머니 방에서 녹음한 음성을 자동으로 기억 갤러리에 저장

**디바이스 등록** (최초 1회, IoT 기기 설정 시)
- 서버에 디바이스 시리얼 `ESP32-001` 등록

**환자 연동**
- 홍보호자가 앱에서 `ESP32-001`을 어머니 이순자에게 연동

**자동 음성 업로드**
- 디바이스가 녹음 후 S3 URL을 서버로 자동 전송
- 기억 갤러리에서 음성 기록 확인 가능
